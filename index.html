Problem 1

Discovery — 63.7% never start

KYC is invisible. No persistent CTA, no urgency signal, no cap framing on the dashboard. Users don't hit a wall — they never find the door.

Problem 2

Intent → Submission — 20% drop off

19,473 companies opened the tab and left. More warm-intent users exist than actual applicants. No re-engagement, no draft signal.

Problem 3

Process quality — 70.1% rejected

Photo mismatches, name errors, undisclosed reviewer criteria. No real-time guidance, no field-level feedback on rejection, full restart required.

P1

Dashboard banner — 5 states (unverified / warm-intent / in-review / rejected / approved)
Saldo card cap chip variant (Option B)
Onboarding checklist KYC anchor item (Option C)
Sidebar "Verifikasi Data" status dot (red / yellow / hidden)

Track 2
Fix the 70% Rejection Rate
For those who submit, make approval more likely. Targets the process quality problems inside the KYC form itself.
1–2 Sprints
P1
Photo upload component redesign — real-time quality validation + inline guide (KTP & Selfie)
📉 Drop-off: Very High Effort: Medium Screen: Verifikasi Identitas — Step 2 Compliance: None (same doc, validated earlier)
Evidence from Report
8.9%of all pending reviewer flags in 12m = "Business photo + proof of ownership doesn't match / unclear" — #1 pending reason
13.7%of all-time formal rejection codes = "Invalid Proof of Ownership" — #1 rejection reason historically
12.4%all-time = "Invalid Business Photo + Invalid Proof of Ownership (both failed)" — #2 rejection reason
Current designVerifikasi Identitas screen shows two empty upload boxes with no guidance, no example, no quality check. Users wait 24h to learn their photo was wrong.
Design Deliverables
Upload component states: empty · uploading · quality-fail (blur/crop/glare) · accepted
Inline example photo (KTP): correct framing illustration, 4 corners visible, no flash
Inline example photo (Selfie): face + KTP beside face, bright environment
Error states: "Foto terlalu buram", "Sudut KTP tidak terlihat", "Wajah tidak terdeteksi"
P1
Rejection state redesign — field-level feedback + deep-link back to failed step
📉 Drop-off: Very High Effort: Medium Screen: Verifikasi Dokumen Usaha — Rejected state Compliance: None
Evidence from Report
49.6%of 12m submitters try 2+ times — identical to all-time 48.9%. Zero improvement in a year. The loop is structural.
9.5%of submitters try 5+ times (~1,699 companies) — clear sign that feedback loop is broken, not just hard
Root causeNo draft state saved on rejection + vague rejection message = full restart, no specific guidance, another 24h wait
Design Deliverables
Rejected state screen: which section failed (red indicator per step in sidebar)
Field-level error within that section: "Foto KTP tidak terbaca — pastikan semua sudut terlihat jelas"
"Perbaiki" CTA deep-links to the exact failed field, all other data pre-filled
Draft indicator: all previously accepted data shown as locked/pre-filled on resubmission
P2
Document upload screens per business type — contextual docs + inline examples
📉 Drop-off: High Effort: Medium Screen: Verifikasi Dokumen Usaha — downstream of type selector Compliance: None
Evidence from Report
5.7%all-time rejection code = "Invalid Business Photo only" — no guidance at upload time
4.6%all-time = "Wrong Business Type Selected" — users pick the wrong type and upload wrong documents
Current designBusiness type selector (Social Media / E-commerce / Perusahaan Jasa / Perusahaan Dagang) exists but the downstream upload screens per type are not yet designed
Design Deliverables
4 upload screen variants (one per business type) with contextual document list
Social Media: screenshot of profile page (follower count visible) + recent post
E-commerce: screenshot of store page (rating + review count visible)
Perusahaan Jasa/Dagang: exterior photo of premises + signage visible
"Why this document?" tooltip on each upload field
P2
Pre-submission name match warning — fuzzy compare form input vs KTP
📉 Drop-off: Medium Effort: Medium Screen: Verifikasi Identitas — Step 2 (Nama field) Compliance: Improves data accuracy
Evidence from Report
7.3%all-time rejection code = "Owner Name Mismatch (KTP name ≠ account name)" — #3 rejection reason
2.1%12m reviewer pending flag = "User name doesn't match KTP" — avoidable with a pre-submission check
6.4%all-time = "Business Name Mismatch" — same pattern on the business name field
Design Deliverables
Inline warning on "Nama Pemilik Bisnis" field when typed name ≠ KTP OCR result
Warning copy: "Nama yang Anda masukkan berbeda dengan nama di KTP. Periksa sebelum melanjutkan."
Two options shown: "Perbaiki nama" or "Lanjutkan (nama KTP berbeda)"
P3
Draft state visibility — auto-save indicator across all 3 KYC steps
📉 Drop-off: Medium Effort: Low Screen: All 3 KYC steps (Informasi / Identitas / Dokumen) Compliance: None
Evidence from Report
20.0%viewed the tab and didn't submit — unknown how many started filling the form and closed mid-way thinking they'd lose data
Benchmark (Flip)Flip auto-saves silently with no "save as draft" button — data never lost on close. This is the target pattern.
Benchmark (Xendit)Xendit loses data on back/leave if form is partial — same broken pattern Paper.id has today.
Design Deliverables
Auto-save timestamp badge near step title: "Tersimpan otomatis: 14.32"
On return to tab: "Lanjutkan dari terakhir kali" banner at top of Step 1
Leave-page confirmation modal: "Data Anda sudah tersimpan. Anda bisa kembali kapan saja."

P1
eKYC vendor integration UI — liveness check screen + sub-5 minute approval state
📉 Drop-off: Very High Effort: High Screen: Verifikasi Identitas — replaces static upload Compliance: OJK POJK 23/2019 compliant
Evidence from Report
70.1%PERORANGAN submissions rejected — 100% go to manual review with 24h SLA. Root fix is automated OCR + liveness.

Compliance noteeKYC liveness (Privy, Verihubs) is actually MORE rigorous than current static selfie review. Compliant and faster.

Sub-5 minute approval state (success screen, different from 24h pending state)
Fallback path: "Verifikasi otomatis gagal → dialihkan ke review manual" state
P2
Risk-tiered graduated transaction limits — new "Stage 1 verified" mid-tier state
📉 Drop-off: Very High Effort: High Screen: Dashboard, PaperPay, all financial screens Compliance: Low risk (OJK simplified CDD)
Evidence from Report
84,358companies stuck at binary Rp 100K cap — either fully blocked or fully unlocked. No in-between state to reward partial progress.
Proposed tiersStage 1 verified → Rp 500K ceiling · KYC complete → standard limit · KYB complete → no limit
OpportunityAn intermediate limit creates a natural conversion trigger — users who hit Rp 500K are motivated to complete full KYC
Design Deliverables
New "Stage 1 verified" state across balance card, PaperPay flow, transaction screens
Cap messaging per tier: Rp 100K / Rp 500K / Standard / Unlimited
Tier upgrade prompt when user hits the active cap mid-transaction
Feature unlock table update (What each tier unlocks)
P2
NPWP + NIB API auto-verification UI — replace legal doc photo upload for KYB
📉 Drop-off: High Effort: High Screen: Verifikasi Dokumen Usaha — PT/CV/Firma/Koperasi Compliance: Stronger than photo review
Evidence from Report
5.4%all-time rejection = "Invalid SK Kemenkumham" — legal entity document wrong or unreadable as a photo
4.8%all-time = "Invalid Deed of Company" — same photo-review quality problem for business entities
KYB note~100% of KYC-approved business entities also complete KYB — KYB itself isn't the drop-off. But the manual doc review still wastes time on clean submissions.
Design Deliverables
Text input for NPWP / NIB number (replaces photo upload)
API lookup loading state + success state (OSS / DJP verified)
Error state: number not found / mismatch — with specific error message and CS escalation path
